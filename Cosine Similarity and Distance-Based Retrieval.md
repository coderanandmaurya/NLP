# 2. Motivation: Why Measure Similarity?

In modern NLP systems, text is converted into **vector representations called embeddings**.

Example:

| Text             | Vector              |
| ---------------- | ------------------- |
| Apple is a fruit | [0.52, 0.11, 0.88]  |
| Mango is a fruit | [0.50, 0.13, 0.86]  |
| Cars run on fuel | [-0.42, 0.71, 0.12] |

Question:

**How do we determine which texts are similar?**

We use a similarity metric such as **cosine similarity**.

Cosine similarity

---

# 3. What is Cosine Similarity?

Cosine similarity measures the **similarity between two vectors by calculating the cosine of the angle between them**.

Formula:

[
\text{Cosine Similarity} =
\frac{A \cdot B}{||A|| \times ||B||}
]

Where:

* (A \cdot B) = dot product of vectors
* (||A||) = magnitude of vector A
* (||B||) = magnitude of vector B

The result ranges between **-1 and 1**.

---

# 4. Interpretation of Cosine Values

| Cosine Value | Angle       | Interpretation    |
| ------------ | ----------- | ----------------- |
| 1            | 0°          | Identical vectors |
| 0.7 – 0.9    | small angle | High similarity   |
| 0            | 90°         | No similarity     |
| -1           | 180°        | Opposite meaning  |

Important idea:

Cosine similarity compares **vector direction rather than vector magnitude**.

---

# 5. Geometric Intuition

Imagine vectors starting from the same origin.

Case 1 – Similar meaning

```
Vector A →
Vector B →
Angle ≈ 0°
Cosine ≈ 1
```

Case 2 – Unrelated

```
Vector A →
Vector B ↑
Angle ≈ 90°
Cosine ≈ 0
```

Case 3 – Opposite meaning

```
Vector A →
Vector B ←
Angle ≈ 180°
Cosine ≈ -1
```

Thus, smaller angle → higher similarity.

# 6. Example with Text Embeddings

Suppose embeddings generated by a model such as:

Word2Vec

Vectors:

| Word  | Vector              |
| ----- | ------------------- |
| king  | [0.23, 0.51, 0.10]  |
| queen | [0.22, 0.50, 0.11]  |
| apple | [-0.31, 0.14, 0.70] |

Similarity results:

| Words        | Cosine Similarity |
| ------------ | ----------------- |

Conclusion:

# 7. Distance-Based Retrieval

Distance-based retrieval means **finding the most similar vectors to a query vector**.

Process:

```
Query text
      ↓
Embedding model
      ↓
Query vector
      ↓
Similarity comparison
      ↓
Retrieve closest vectors
```

This is the foundation of **semantic search systems**.

---

# 8. Example of Distance-Based Retrieval

Query:

```
healthy fruits
```

Database of embeddings:

| Document              | Similarity Score |
| --------------------- | ---------------- |
| benefits of apple     | 0.92             |
| fruit nutrition guide | 0.88             |
| smartphone review     | 0.11             |

System returns **top similar documents**.

---

# 9. Types of Distance Metrics

While cosine similarity is most common in NLP, other distance metrics exist:

| Metric             | Description                            |
| ------------------ | -------------------------------------- |
| Cosine Similarity  | compares vector direction              |
| Euclidean Distance | straight-line distance between vectors |
| Manhattan Distance | sum of absolute differences            |

In NLP and embeddings, **cosine similarity is preferred** because it ignores vector magnitude.

Distance-based retrieval is used in many AI systems:

# 11. Practical Pipeline

A typical semantic retrieval system works as follows:

```
Text data
   ↓
Embedding model
   ↓
Vector representations
   ↓
Vector database
   ↓
Cosine similarity search
   ↓
Top-k relevant results
```

Vector databases enable fast retrieval in large datasets.

---

# 12. Key Takeaways

* Cosine similarity measures **similarity between vectors using angle**.
* Values range from **-1 to 1**.
* Higher value indicates **greater semantic similarity**.
* Distance-based retrieval finds **nearest vectors to a query**.
* This approach powers **semantic search, recommendation systems, and generative AI applications**.
